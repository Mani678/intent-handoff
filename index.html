⚡ Intent Handoff System

Most agent systems pass state between agents. This passes intent — and tracks how it degrades with every hop. Two demos: handoff decay and race condition resolution.

HOW IT WORKS

INTENT ENVELOPE
Every task carries a "why" — the original reason. This travels with the task across agents.
CONFIDENCE DECAY
Each hop reduces confidence by 12%. More agents = more dilution of original intent.
REANCHOR THRESHOLD
Below 0.40 confidence, the system routes to a human instead of blindly executing.
MAX HOPS
After 5 hops the task aborts and escalates regardless of confidence.

STATUS LEGEND

Healthy — proceed normally
Degraded — proceed with caution
Needs reanchor — route to human
Max hops exceeded — abort

THE RACE CONDITION PROBLEM

THE PROBLEM
Two agents claim the same intent simultaneously. Without a lease system, both execute — causing duplicate or conflicting actions.
THE FIX — INTENT LEASE
First agent to claim wins a time-bounded lease. Others are denied gracefully. If the winner fails, the lease expires and another agent can claim.
LEASE STATES
Unclaimed → Leased → Completed (or Expired). Decay tracker runs independently — even a valid lease gets aborted if intent degrades.
WHY THIS MATTERS
Most handoff systems ignore concurrent claims. This is the real edge case that causes silent failures in production agent systems.
